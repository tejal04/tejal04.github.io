Image Caption Generation

Image captioning, a pivotal intersection of Computer Vision and Natural Language Processing (NLP), involves the transformation of visual content into concise textual descriptions. This research undertaking aimed to assess the effectiveness of three distinct architectural approaches in addressing this intricate challenge: Encoder-Decoder / Inject Architecture, Encoder-Decoder with Attention Architecture, and Merge / Multi-Modal Architecture.

Dataset :
Flickr8k dataset consists of 8,096 images. These images cover a wide range of subjects, scenes, and objects, making it a diverse collection for research purposes.

Machine Learning :
The Image Captioning System is made up of three main parts: the Image Encoder, the Sequence Decoder, and the Sentence Generator. These 3 components are connected in obtain different architectures that are explored in this project - Encoder-Decoder / Inject Architecture, Encoder-Decoder with Attention Architecture, and Merge / Multi-Modal Architecture. BLEU (Bilingual Evaluation Understudy) are used to measure the quality of generated captions. BLEU compares the generated captions to reference captions to assess how well they align with human-generated descriptions.

Tech Stack :
tensorflow, keras, python, pandas, numpy.

Virtual Stock

Stock Trading Simulation Desktop Application

The "Virtual Stock" project represents a Java Swing Model-View-Controller (MVC) application designed to simulate stock trading with a keen focus on best practices and optimized system architecture. An application that helps users who are new to investing to learn about how money could grow, in the style of "virtual gambling". Similar to virtual gambling, our application will not use real money and investment. Instead it will allow the user to create investment portfolios, try out buying and selling of stock, and various investment strategies to see how they can grow (or shrink) their money with time.

Architecture :
The system follows a robust Model-View-Controller (MVC) architecture, ensuring a clear separation of components and promoting maintainability. The Model encapsulates the application's data and logic, the View handles the user interface, and the Controller manages user interactions, creating a highly modular and extensible system.

Web API Integration :
The application is integrated with a web API to provide real-time stock data, enhancing the simulation's authenticity. This integration not only enriches the user experience but also allows for efficient data storage and retrieval, ensuring that users have access to the latest market information.

Design Patterns :
The project incorporates various design patterns to optimize its architecture. Patterns like Singleton, Abstract Factory, and Factory Method have been utilized to ensure that each component is highly maintainable and extensible.

Tech Stack :
Java, API, MVC, Java Swing, Git.
